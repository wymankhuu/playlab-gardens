All databases live under the **Playlab Community Gardens** page (`328a9d37-78c5-8080-a335-ec202b497c01`).

## Data Files (data/)

Generated by `node scripts/export-notion.js`. Do not hand-edit ‚Äî changes will be overwritten.

| File | Contents |
|------|----------|
| `collections.json` | All apps grouped by collection tag. Each collection has `id`, `name`, `type`, `iconColor`, `appCount`, and an `apps` array. Seed collections are appended at the end. |
| `seeds.json` | Seed templates with `seeds` (flat array) and `collections` (grouped by Seed Collection field). |
| `cultivators.json` | Cultivator profiles with matched apps (by creator name lookup against collections.json). |

## Export Script

```bash
# Requires .env with NOTION_API_KEY and NOTION_MASTER_DB_ID
node scripts/export-notion.js
```

The script:
1. Queries the Master DB (paginates through all rows)
2. Parses each row into an app object (name, URL, creator, description, usage, impact, sessions, iterations, tags)
3. Groups apps by Collection multi-select tags
4. Deduplicates by app ID and name (keeps the entry with the longest content)
5. Sorts apps by sessions descending within each collection
6. Queries the Seeds DB and appends seed collections
7. Queries the Cultivators DB and matches profiles to apps
8. Writes all three JSON files

## API Layer (api/)

Serverless functions deployed on Vercel. Read from the JSON data files at runtime.

| Endpoint | Purpose |
|----------|---------|
| `GET /api/collections` | Returns all collections (without full app arrays ‚Äî just metadata + preview apps) |
| `GET /api/collection/:id` | Returns a single collection with all its apps |
| `GET /api/search?q=` | Full-text search across app names, descriptions, and creators |
| `GET /api/app-labels` | Returns admin labels/overrides from the Overrides DB |
| `POST /api/submit-app` | App submission form ‚Üí creates entry in Master DB + Slack notification |
| `POST /api/admin-save` | Admin panel ‚Üí updates app fields in Master DB |
| `POST /api/admin-pin` | Admin panel ‚Üí toggles homepage pin flag |
| `POST /api/star` | Star/unstar an app (Upstash Redis counter) |
| `GET /api/stars?ids=` | Bulk fetch star counts |

## Environment Variables

See `.env.example` for the full list. Key groups:

- **Notion**: `NOTION_API_KEY`, `NOTION_MASTER_DB_ID` ‚Äî required for export script and all write APIs
- **Admin**: `ADMIN_PASSWORD` ‚Äî protects the admin edit/pin endpoints
- **Redis**: `KV_REST_API_URL`, `KV_REST_API_TOKEN` ‚Äî powers star counts (Upstash via Vercel Marketplace)
- **Slack**: `SLACK_WEBHOOK_URL` ‚Äî optional notification on app submission
